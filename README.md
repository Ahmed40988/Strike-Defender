# Strike Defender 🛡️

Strike Defender is a graduation project backend API built with **ASP.NET Core** following **Clean Architecture** principles.  
The system focuses on security, authentication, and scalable backend design.

---

## 🎓 Graduation Project Overview
Strike Defender aims to provide a secure and well-structured backend foundation with modern authentication mechanisms, clean separation of concerns, and reusable core services.

---

## 🧱 Architecture
The project follows **Clean Architecture**, divided into the following layers:

- **API Layer** – Controllers, request handling, and middleware
- **Application Layer** – Use cases, commands, validation, and business logic
- **Domain Layer** – Core entities, value objects, and domain errors
- **Infrastructure Layer** – Database, Identity, email, and external services

---

## 🛠️ Tech Stack
- ASP.NET Core
- Clean Architecture
- MediatR
- FluentValidation
- ErrorOr
- Entity Framework Core
- ASP.NET Core Identity
- JWT Authentication & Refresh Tokens

---

## 🚀 Features
- User registration and login
- Email confirmation with OTP
- Forgot and reset password
- JWT authentication with refresh tokens
- Role-based access support
- Centralized validation and error handling
- Reusable infrastructure services

---

## 📦 Core Services
- Email Service
- Fuzzy Search Service
- Pagination Helper
- File Handling Utilities

---

## ▶️ Getting Started
1. Clone the repository
2. Configure `appsettings.json`
3. Run database migrations
4. Start the application
